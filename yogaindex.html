1. Pengertian OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek. Objek dibuat dari sebuah kelas (class) yang berisi atribut (data) dan method (fungsi).

2. Perbedaan Kelas dan Objek

Kelas (Class) adalah blueprint atau cetak biru untuk membuat objek. Kelas mendefinisikan atribut (data) dan metode (perilaku).

Objek (Object) adalah instance (wujud nyata) dari kelas. Objek memiliki nilai nyata untuk atribut dan dapat menjalankan metode.

Contoh:

  • Hewan = kelas
  • Kucing, Anjing = objek

3. Tiga Keuntungan OOP

  • Reusability: Kode dapat digunakan kembali melalui pewarisan.
  • Encapsulation: Data lebih aman karena dibatasi aksesnya.
  • Modularity: Program lebih terstruktur dan mudah dikelola.

Manfaat tambahan:

  • Mudah dikembangkan
  • Lebih mudah debugging
  • Cocok untuk program besar

4. Contoh Class dan Object (Dart)


class Car {
  String model;

  Car(this.model);

  void start() {
    print("Mobil $model dinyalakan!");
  }
}

void main() {
  var mobil = Car("Toyota");
  mobil.start();
}

5. Constructor & Validasi


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError("IPK harus 0.0 - 4.0");
    }
  }
}

6. Encapsulation


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  String get name => _name;
  set name(String value) => _name = value;
}

7. Contoh Kelas Hewan


class Hewan {
  String nama;

  Hewan(this.nama);

  void bersuara() {
    print("$nama mengeluarkan suara!");
  }
}

void main() {
  var kucing = Hewan("Kucing");
  kucing.bersuara();
}

Diskusi: Kapan Menggunakan OOP?

OOP sebaiknya digunakan ketika membuat program yang cukup besar atau kompleks, misalnya aplikasi sekolah, toko online, atau game. OOP sangat cocok karena konsepnya meniru objek di dunia nyata sehingga memudahkan pengelolaan entitas seperti pengguna atau produk.

Kesimpulan

OOP mempermudah pengelolaan program besar karena kode lebih terstruktur, mudah dikembangkan, dan lebih aman melalui konsep enkapsulasi.
