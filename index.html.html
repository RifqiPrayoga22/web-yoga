1. Pengertian OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek. Objek dibuat dari sebuah kelas (class) yang berisi atribut (data) dan method (fungsi).

Konsep utama OOP meliputi:

  • Class dan Object
  • Encapsulation (Enkapsulasi)
  • Constructor
  • Method

2. Class & Object

Kelas (class) adalah blueprint atau cetak biru untuk membuat objek, sedangkan objek (object) adalah instance atau wujud nyata dari kelas tersebut[cite: 10, 11].

class Car {
  String? model;
  String? color;

  void start() {
    print('Mobil $model berwarna $color dinyalakan!');
  }
}

void main() {
  var mobil = Car();
  mobil.model = "Toyota";
  mobil.color = "Hitam";
  mobil.start();
}

3. Constructor & Validasi

Constructor digunakan untuk menginisialisasi nilai atribut saat objek dibuat. Validasi dalam constructor memastikan data yang dimasukkan memenuhi kriteria tertentu.

class Mahasiswa {
  String nama;
  double ipk;

  Mahasiswa(this.nama, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError("IPK harus berada dalam rentang 0.0 - 4.0");
    }
  }
}

4. Encapsulation (Enkapsulasi)

Enkapsulasi membatasi akses langsung ke atribut menggunakan setter dan getter agar data lebih aman.

class Person {
  String _name; // Private attribute
  int age;

  Person(this._name, this.age);

  String get name => _name;
  set name(String value) => _name = value;
}

5. Perbedaan Kelas & Objek serta Keuntungan OOP

Perbedaan Utama:

  • Kelas: Cetak biru yang mendefinisikan atribut (data) dan metode (perilaku)[cite: 10]. Contoh: Kelas Hewan[cite: 13].
  • Objek: Instance nyata yang memiliki nilai atribut dan menjalankan metode[cite: 11]. Contoh: kucing, anjing[cite: 14].

Keuntungan OOP:

  • Reusability: Kode dapat digunakan kembali melalui pewarisan[cite: 16].
  • Encapsulation: Data lebih aman karena dibatasi aksesnya[cite: 17].
  • Modularity: Program dibagi menjadi bagian kecil sehingga mudah dikelola[cite: 18].
  • Manfaat tambahan: Lebih mudah dikembangkan, debugging lebih mudah, dan cocok untuk program besar[cite: 19, 20, 21, 22].

6. Contoh Implementasi Kelas Hewan (Dart)

Berikut adalah implementasi kelas Hewan (dikonversi dari contoh Python ke Dart sesuai tema laporan)[cite: 23, 24]:

class Hewan {
  String nama;
  Hewan(this.nama);

  void bersuara() {
    print("$nama mengeluarkan suara!");
  }
}

void main() {
  var kucing = Hewan("Kucing");
  kucing.bersuara();
}

7. Diskusi Terbuka: Kapan Menggunakan OOP?

OOP sebaiknya digunakan saat membangun program yang cukup besar atau kompleks, seperti aplikasi sekolah, toko online, atau game[cite: 34]. OOP sangat cocok karena konsepnya meniru objek di dunia nyata, memudahkan pengelolaan entitas seperti pengguna atau produk[cite: 35].

Kesimpulan

OOP mempermudah pengelolaan program besar karena kode lebih terstruktur, mudah dikembangkan, dan lebih aman melalui konsep enkapsulasi.
